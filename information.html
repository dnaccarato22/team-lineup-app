    How Smart Lineup Generator Works

    How Lineup Generation Works

    What the app looks at when building a lineup for your game.

    1. The app starts with the roster you selected on the Lineup page, along with the game date and each player's available innings.
    2. It looks at every player's position scores to understand where they prefer to play.
    3. It start by assigning the game's pitchers. Players with higher pitching scores will be treated as starters, while lower scores will be treated as relievers.
    4. It balances field assignments across innings so players are best placed where they prefer while still respecting even playing time.
    5. It assigns a batting order according to the most recent lineup, but all of the positions and batting spots can be reviewed and edited.
    6. If you make manual lineup edits, the app sends the updated lineup back to the API for validation before saving it.
    What You Can Customize

    The main inputs that affect the generated lineup.

    Roster Selection

    Only players added to the roster are considered for the generated lineup.

    Game Date

    The Lineup page defaults to the upcoming Saturday, but you can change the date before generating a lineup.

    Available Innings

    Each player can be marked for the full game or partial innings if they are arriving late or leaving early.

    Manual Lineup Editing

    After generation, you can drag players to change batting order and update inning positions before saving.

    How To Manage Players

    Best practices for keeping the player directory accurate.

    1. Open the Players page to add new players, update existing ratings, or remove players who are no longer active.
    2. When adding or editing a player, fill out each position score based on where they prefer to play.
    3. Use the search field to quickly find players by first or last name.
    Position Scores

    What the position ratings mean for Players.

    Score Range

    Position scores run from 0 to 5, where higher values are the highest preference.

    0
    No Assignment

    A player will never be assigned to this position.

    1
    Emergency Option

    The player is willing to play this position only if no one else is available.

    2
    Somewhat Comfortable

    The player can handle the position, but it is not a strong preference.

    3
    Comfortable

    The player is comfortable here and should be a dependable option.

    4
    Very Comfortable

    The player is a strong fit for this spot and is usually a good choice there.

    5
    Top Preference

    This is the player's most preferred position and where they should thrive most often.
